# 開発依頼書：楽天ランキングAPI × Google Apps Script

**作成日：** 2026年（随時更新）

---

## 1. プロジェクト概要

楽天市場のランキングAPIを使用して、複数カテゴリのランキング情報を定期的に取得し、Google スプレッドシートへ保存するGoogle Apps Script（GAS）を開発する。
将来的にはSupabaseへの保存機能も追加する。

---

## 2. 使用するAPI

### 楽天市場 商品ランキングAPI

| 項目 | 内容 |
|---|---|
| APIバージョン | 2022-06-01 |
| エンドポイント | `https://openapi.rakuten.co.jp/ichibaranking/api/IchibaItem/Ranking/20220601` |
| 公式ドキュメント | https://webservice.rakuten.co.jp/documentation/ichiba-item-ranking |
| 認証方式 | `applicationId`（必須）＋ `accessKey`（必須） |

### 主要リクエストパラメータ

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| `applicationId` | String | ✅ | アプリケーションID |
| `accessKey` | String | ✅ | アクセスキー |
| `genreId` | long | - | ジャンルID（genreIdとage/sexは併用不可） |
| `age` | int | - | 年代（10/20/30/40/50） |
| `sex` | int | - | 性別（0:男性 / 1:女性） |
| `page` | int | - | ページ番号（1〜34、1ページ30件） |
| `period` | string | - | `realtime` を指定するとリアルタイムデータ |
| `elements` | String | - | 返却フィールドを絞り込む（カンマ区切り） |
| `formatVersion` | int | - | `2` を推奨（シンプルな配列形式） |
| `carrier` | int | - | 0:PC / 1:モバイル（デフォルト:0） |

### 主要レスポンスフィールド

| フィールド | 説明 |
|---|---|
| `rank` | 順位 |
| `itemName` | 商品名 |
| `catchcopy` | キャッチコピー |
| `itemCode` | 商品コード |
| `itemPrice` | 価格 |
| `itemUrl` | 商品URL |
| `smallImageUrls` | 商品画像URL（64x64、最大3枚） |
| `mediumImageUrls` | 商品画像URL（128x128、最大3枚） |
| `reviewCount` | レビュー数 |
| `reviewAverage` | レビュー平均点 |
| `shopName` | ショップ名 |
| `shopUrl` | ショップURL |
| `availability` | 在庫フラグ（0:在庫なし / 1:在庫あり） |
| `genreId` | ジャンルID |

### APIの制約・注意事項

- `genreId` と `age` / `sex` は同時指定不可
- `period=realtime` は `age` パラメータとの組み合わせ時のみ使用可能
- 1ページあたり30件返却。1〜30位はpage=1で取得
- 順位を絞りたい場合はGAS側でレスポンスをスライス処理する
- 短時間に同一URLへの大量アクセスで一時的に応答不能になる場合がある（テスト時に注意）
- ランキング件数がAPIとランキングサイトで一致しない場合がある（削除済み商品はスキップされるため）

## 3. システム構成

```
[スクリプトプロパティ]
  ・RAKUTEN_APPLICATION_ID
  ・RAKUTEN_ACCESS_KEY
        ↓
[設定シート（手動編集）]
  ・ジャンル名 / ジャンルID / 取得件数 / 有効フラグ
        ↓
[GASメイン処理]
  ① 設定シートを読み込む
  ② 有効フラグがTRUEのジャンルをループ
  ③ 楽天ランキングAPIを呼び出す
  ④ 指定件数分をランキングシートへ書き込む
        ↓
[ランキングシート（ジャンルごとに自動生成 or 上書き）]
  取得日時 / ジャンル名 / 順位 / 商品名 / 価格 / URL / レビュー数 / 評点 / ショップ名
```

| # | タスク | 詳細 |
|---|---|---|
| 1-1 | スクリプトプロパティ設定 | `RAKUTEN_APPLICATION_ID` と `RAKUTEN_ACCESS_KEY` を安全に保存する関数を作成 |

| # | タスク | 詳細 |

| 2-3 | シート管理 | カテゴリごとにシートを自動生成 or 最新データで上書き更新する |

### Phase 3：運用・安定化

| # | タスク | 詳細 |

| 3-1 | エラーハンドリング | API失敗時のリトライ・エラーログ記録 |
| 3-2 | トリガー設定 | 時間指定の定期実行を設定する関数 |
| 3-3 | 実行ログシート | 実行日時・成否・取得件数を記録する |

### Phase 4：Supabase連携（将来）

| # | タスク | 詳細 |
|---|---|---|
| 4-1 | Supabase接続設定 | URLとAPIキーをスクリプトプロパティへ追加 |
| 4-2 | Supabase書き込み関数 | スプレッドシート書き込みと並行してUpsert処理 |
| 4-3 | テーブル設計 | ランキングデータのスキーマ定義 |

## 7. コーディング規約

- **言語：** Google Apps Script（JavaScript ES6相当）
- **実行環境：** Google Apps Script エディタ（V8ランタイム）
- **文字コード：** UTF-8
- **各関数のヘッダーには必ず説明コメントを記載する**
- **秘匿情報は必ずスクリプトプロパティから取得する**
- **修正時は既存の説明コメントを可能な限り残す・更新する**
- **2案以上ある場合は先に案を提示してオーナーの判断を仰ぐ**
- **コーディング前には必ず方針をオーナーに確認する**

---

## 8. 開発スタイル

- **手法：** アジャイル開発・スモールステップ
- **進め方：** フェーズごとに許可を得てからコーディングを開始する
- **疑問点：** わずかな疑問点でも必ずオーナーに確認してから進める
- **コード修正：** 変更箇所はdiff形式（何行目を修正）で提示する。全行が必要な場合はオーナーから改めて依頼する
